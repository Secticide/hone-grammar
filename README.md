# hone-grammar

Tree-sitter grammar for the [Hone programming language](https://github.com/Secticide/hone).

Used by the [hone-zed](https://github.com/Secticide/hone-zed) Zed editor extension to
provide syntax highlighting, bracket matching, and indentation rules.

---

## Repository structure

```
grammar.js          — grammar definition (edit this)
queries/
  highlights.scm    — syntax highlighting rules
  brackets.scm      — bracket pair definitions
  indents.scm       — auto-indentation rules
src/                — generated by tree-sitter; do not edit by hand
  parser.c
  node-types.json
  grammar.json
  tree_sitter/
package.json
```

---

## Generating the parser

After editing `grammar.js`, regenerate the parser:

```sh
npm install          # first time only
npm run generate     # runs tree-sitter generate
```

This rewrites the files under `src/`. Commit both `grammar.js` and the updated `src/` together.

---

## Relationship with hone-zed

`hone-zed`'s `extension.toml` pins this repository to a specific commit:

```toml
[grammars.hone]
repository = "https://github.com/Secticide/hone-grammar"
commit = "<sha>"
```

When Zed installs the extension it fetches that commit, compiles `src/parser.c` to
WebAssembly internally, and uses the result for parsing. The extension does **not** use a
pre-built binary — the `src/parser.c` committed here is the compilation input.

The query files (`highlights.scm`, `brackets.scm`, `indents.scm`) are **duplicated** into
`hone-zed/languages/hone/` because Zed loads them from the extension repository, not the
grammar repository. Keep the two copies in sync whenever a query file changes.

`outline.scm` lives **only** in `hone-zed` — it is Zed-specific (powers the symbol
outline panel) and has no equivalent in a standard tree-sitter grammar.

---

## Update workflow

1. Edit `grammar.js` and/or files in `queries/`.
2. Run `npm run generate` and verify the parser compiles cleanly.
3. Commit all changes (`grammar.js`, `src/`, and any edited `queries/*.scm`).
4. In `hone-zed`, update the `commit` hash in `extension.toml` to the new SHA.
5. Copy any changed `queries/*.scm` files into `hone-zed/languages/hone/`.
6. Reload the extension in Zed (`zed: reload extensions`).

---

## Known limitations

Nested block comments (`/* /* */ */`) are a Hone language feature that cannot be
expressed as a pure regex token in tree-sitter. The current grammar handles the common
case; an external scanner would be needed to handle arbitrary nesting depth.
